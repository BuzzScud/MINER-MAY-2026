# PHP Extension Build Troubleshooting Guide

## Problem
The PHP extension build fails with:
```
/usr/bin/ld: cannot find .libs/crystalline_math_extension.o: No such file or directory
```

## Root Cause
The Makefile generated by `phpize` and `configure` contains **hardcoded absolute paths** from the environment where it was generated. When you build on a different system (e.g., `/home/logan/code/C/crystalline` instead of `/workspace`), these paths are incorrect and the compilation step is skipped.

## Solution

### Option 1: Use the Rebuild Scripts (RECOMMENDED)

I've created rebuild scripts that will properly regenerate the build configuration for your system:

```bash
# For math extension
cd php/math
./rebuild.sh

# For algorithms extension
cd php/algorithms
./rebuild.sh
```

### Option 2: Manual Rebuild

If you prefer to do it manually:

```bash
# Clean everything first
make clean

# For math extension
cd php/math
make clean 2>/dev/null || true
phpize --clean
rm -rf .libs *.lo *.la modules/ autom4te.cache/
phpize
./configure
make

# For algorithms extension
cd ../algorithms
make clean 2>/dev/null || true
phpize --clean
rm -rf .libs *.lo *.la modules/ autom4te.cache/
phpize
./configure
make

# Return to root and build everything
cd ../..
make
```

### Option 3: Build Without PHP Extensions

If you don't need the PHP extensions, you can build just the core libraries:

```bash
make clean
make math
make algorithms
make cllm
```

## What I Fixed

### 1. Enhanced `php-clean` Target
The `php-clean` target now:
- Properly cleans the Makefile
- Runs `phpize --clean` to remove generated files
- Removes `.libs/`, `*.lo`, `*.la`, `modules/`, and `autom4te.cache/` directories
- Handles errors gracefully

### 2. Integrated PHP Cleaning into Main Clean
The main `clean` target now calls `php-clean` automatically, ensuring PHP extensions are properly cleaned when you run `make clean`.

### 3. Created Rebuild Scripts
Two scripts (`php/math/rebuild.sh` and `php/algorithms/rebuild.sh`) that automate the entire rebuild process.

## Why This Happens

When you run `phpize` and `./configure`, they generate a Makefile with absolute paths based on your current directory. For example:

```makefile
crystalline_math_extension.lo: /workspace/php/math/crystalline_math_extension.c
```

If you're building in `/home/logan/code/C/crystalline`, this path is wrong, and Make skips the compilation step because it can't find the source file.

## Prevention

To avoid this issue in the future:

1. **Always regenerate build files** when:
   - Cloning the repository to a new location
   - Moving the project directory
   - Switching between different systems

2. **Use the rebuild scripts** instead of running `make` directly for PHP extensions

3. **Run `make clean`** before building after any configuration changes

## Verification

After rebuilding, verify the build succeeded:

```bash
# Check that extensions were built
ls -lh php/math/modules/crystalline_math.so
ls -lh php/algorithms/modules/crystalline_algorithms.so

# Verify no warnings or errors
make clean && make 2>&1 | grep -i "error\|warning" | grep -v "configure.ac.*obsolete"
```

You should see:
- Both `.so` files exist
- No errors or warnings (except harmless configure.ac warnings)

## Additional Notes

### PHP Version Compatibility
The extensions are built for the PHP version installed on your system. If you see warnings about PHP API version mismatches, make sure you're using a compatible PHP version (7.4+ recommended).

### Required Tools
Make sure you have these tools installed:
- `phpize` (from php-devel or php-dev package)
- `php-config`
- `gcc` or `cc`
- `make`
- `autoconf`
- `libtool`

### Installation
After building, you can install the extensions:
```bash
sudo make install-php
```

Then add to your `php.ini`:
```ini
extension=crystalline_math.so
extension=crystalline_algorithms.so
```

## Summary

The key issue is that **generated Makefiles contain absolute paths**. The solution is to **regenerate them on your system** using the provided rebuild scripts or by running `phpize --clean && phpize && ./configure` in each PHP extension directory.

The enhanced `make clean` target now handles this automatically, ensuring a clean slate for rebuilding.
